Build a floating AI Command Dialog accessible from every single page of ArgiCRM. This is a single, always-available window where the user can type any instruction in plain language and the AI agent will either execute it directly or dispatch the instruction to the correct module across the platform — without the user ever leaving their current page.
Think of it as the brain of ArgiCRM — one window to control everything.

⚙️ TECHNICAL ARCHITECTURE
AI Engine

Use Anthropic Claude API (claude-sonnet-4-20250514) as the brain
System prompt must define ARIA's identity, platform context, and full module capability map
Maintain conversation history within the session so ARIA remembers context across multiple instructions in the same dialog session
If the user's instruction is ambiguous, ARIA asks one clarifying question before acting

Intent Router
Build an Intent Router on the backend that:

Receives ARIA's parsed instruction
Identifies which module and action is targeted
Calls the correct internal API endpoint to execute the action
Returns a confirmation or result back to the dialog

Intent categories to handle:
CREATE · READ · UPDATE · DELETE · EXPORT · SEND · SCHEDULE · SUMMARIZE · NAVIGATE · CONNECT
Execution vs Instruction Mode
ARIA operates in two modes:

EXECUTE MODE — ARIA directly performs the action via API call (e.g. creates a contact, adds a product, generates an invoice). Confirm with: "Done — contact John Doe has been added to your CRM."
INSTRUCTION MODE — For complex or destructive actions (e.g. delete all contacts, send mass email), ARIA describes exactly what it will do and asks: "Should I go ahead?" before executing.

Security

All ARIA actions are authenticated via the logged-in user's JWT token
ARIA can only act within the permissions of the current user's role
Log every ARIA action to an audit trail in the database (timestamp, user, instruction, action taken, module affected)
ARIA must never expose other users' data


🧠 ARIA SYSTEM PROMPT (pass this to Claude API)
You are ARIA, the AI Command Agent embedded inside ArgiCRM, 
a CRM and business management platform built by ARGILETTE LLC.

Your job is to understand the user's plain-language instructions 
and execute them across the ArgiCRM platform — including CRM, 
E-Commerce, Projects, Billing, Marketing, and Account modules.

Rules:
- Always respond in the same language the user writes in (EN or FR)
- Be concise and action-oriented — confirm what you did, not just what you will do
- For destructive or irreversible actions, always confirm before executing
- If an instruction is unclear, ask exactly ONE clarifying question
- Never make up data — if something doesn't exist, say so clearly
- Always tell the user which module you are acting on
- If a task spans multiple modules, handle them in logical order and report each step

You have access to the following platform actions:
CRM: create/read/update/delete contacts, leads, deals, pipelines
E-Commerce: create stores, add products, manage orders, link domains
Projects: create/manage projects and tasks
Billing: create invoices, track payments, connect payment accounts
Marketing: draft campaigns, schedule posts, manage sequences
Settings: manage account, team members, integrations

Current user context will be injected with each request.
