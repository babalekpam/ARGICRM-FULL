STEP 1 — BUILD THE DATA INGESTION ENGINE
Build a backend service that pulls data from these sources automatically on a scheduled cron job (daily refresh):
Source 1 — NPI Registry (US Healthcare)
Endpoint: https://npiregistry.cms.hhs.gov/api/?version=2.1
No API key required
Pull: provider name, practice address, phone, specialty, NPI number, entity type
Target volume: 500,000+ records
Schedule: weekly refresh
Source 2 — Google Places API (US & Africa Businesses)
Requires: Google Cloud account (free $200 credit/month)
Pull: business name, address, phone, website, category, rating, hours
Target: search by category + city across US and African cities
Schedule: daily for new businesses
Source 3 — Yelp Fusion API (US Businesses)
Requires: free Yelp developer account
Pull: business name, address, phone, category, rating, review count
Schedule: weekly refresh
Source 4 — OpenCorporates (Company Data)
Endpoint: https://api.opencorporates.com
No key required for basic queries
Pull: company name, jurisdiction, registered address, status, officers
Schedule: weekly refresh
Source 5 — OpenStreetMap / Overpass API (Africa)
Endpoint: https://overpass-api.de/api/interpreter
No account required
Pull: business name, address, category, coordinates across Togo, Ghana, Nigeria, Senegal, Côte d'Ivoire
Schedule: weekly refresh
Source 6 — CMS Medicare Data (US Healthcare)
Endpoint: https://data.cms.gov/provider-data
No key required
Pull: provider name, specialty, location, Medicare participation status
Schedule: monthly refresh
For each source build:

A dedicated fetcher service file
Error handling and retry logic
Rate limit management to stay within free tiers
A normalization function that maps each source's fields to a unified schema


STEP 2 — UNIFIED DATABASE SCHEMA
Create a single leads table in PostgreSQL with this unified schema:
id, source, market (US/Africa), category, full_name, 
company_name, title, email, phone, website, address, 
city, state, country, zip, industry, specialty, 
employee_size, revenue_range, rating, linkedin_url, 
language (EN/FR), verified (boolean), last_updated, 
times_sold, available (boolean)
Also create:

data_packages table — predefined lead list bundles clients can buy
client_purchases table — tracks which client bought which leads
lead_credits table — tracks each client's remaining lead credits
ingestion_logs table — logs every data pull (source, count, timestamp, errors)

Rules:

Deduplicate on phone + company name combination
Mark a lead available: false once sold to 3 different clients maximum
Never sell the same lead to the same client twice

STEP 5 — AUTOMATED ENRICHMENT (ZERO COST)
After ingestion, run an enrichment pass using:
Email Finder:

Use Hunter.io free tier (25/month) for high-value leads only
Pattern-match common email formats (firstname.lastname@domain.com) for the rest
Validate email format before storing

LinkedIn URL:

Construct LinkedIn search URL from name + company
Store as suggested LinkedIn URL (not verified — flag as unverified)

AI Cleaning (Claude API):

Pass raw scraped records through Claude to:

Normalize inconsistent formatting
Fill in missing fields where inferable
Translate French records to English and vice versa
Score lead quality 1–10 based on data completeness



Use this Claude prompt for cleaning:
You are a data quality agent. Given this raw business record, 
normalize all fields, fix formatting inconsistencies, infer 
any missing fields where possible, translate if needed 
(preserve both EN and FR versions), and return a clean JSON 
object matching the unified lead schema. Score data completeness 
from 1-10. Flag any suspicious or invalid records.
